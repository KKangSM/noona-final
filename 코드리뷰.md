## 2. 아키텍처 & 요청 처리 흐름

```
[브라우저]
   │  HTTP 요청 (예: GET /products/:id)
   ▼
server.js            진입점 — API 라우팅 or 정적 파일
   │  router.handle(req, res)
   ▼
backend/router.js    "METHOD URL" → 컨트롤러 매핑 (:id 파라미터 지원)
   │                 DomainError → 상태코드 / 그 외 → 500
   ▼
*.controller.js      요청 파싱 · 응답 (sendJson / readBody)
   │
   ▼
*.service.js         비즈니스 로직 · 검증 (DomainError 던짐)
   │
   ▼
*.repository.js      MongoDB 접근 (findByEmail, findAll, insert ...)
   │
   ▼
[MongoDB]            컬렉션: users · products
```

`*.entity.js` 는 데이터 모델(필드·상태 enum·팩토리)을 정의하고 각 계층이 공유한다.

## 4. 도메인별 엔드포인트

### auth (인증)
- `POST /auth/signup` — 회원가입 (Create)
- `POST /auth/login` — 로그인
- `POST /auth/checkId` — 이메일 중복확인 → `{ available }`

### mypage (내정보 · 회원 R/U/D)
- `GET  /mypage?email=` — 내정보 조회 (Read)
- `POST /mypage/edit` — 이름 수정 (Update)
- `POST /mypage/changePassword` — 비밀번호 변경 (Update)
- `POST /mypage/withdraw` — 회원 탈퇴 (soft-delete: status→WITHDRAWN)

### product (상품 CRUD)
- `GET  /products` — 목록 (list)
- `GET  /products/:id` — 상세 (detail)
- `POST /products/register` — 등록 (register)
- `POST /products/:id/edit` — 수정 (edit)
- `POST /products/:id/delete` — 삭제 (delete)

> K-Evolution 규칙: 리소스 소문자 · 다단어 액션 camelCase · 상태변경은 POST · "URL 마지막 동사 = 핸들러명".

## 5. 프론트엔드 화면 & 연결

| 파일 | 역할 |
|------|------|
| `script.js` | 스플래시(심박선→pen) · 인트로 문구 애니메이션 |
| `auth.js` | 로그인/회원가입(`#auth`) · 내정보(`#mypage`) — `/auth/*`, `/mypage/*` 호출, 세션 localStorage |
| `products.js` | shop 그리드 `GET /products` 렌더 · 카드 클릭 → 상세(`#detail`) `GET /products/:id` |
| `custom.js` | Three.js 3D 커스텀 펜 · 장바구니(localStorage) — `window.PenCart`로 공개 |

**화면 흐름**
```
splash → intro → shop(상품목록) ─┬─ 커스텀 볼펜(3D)  → 장바구니
                                 ├─ 상품 카드 → 상세 → 장바구니
                                 └─ 👤 → 로그인/가입 → 내정보
```

**장바구니 공유**: `custom.js` 가 `window.PenCart = { add, open, render }` 를 노출 →
커스텀 펜과 일반 상품을 같은 카트에 담음. 배지 3곳(shop·custom·detail) 동기화.
